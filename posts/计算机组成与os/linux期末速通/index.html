初步重点

Shell基本用法

基本操作命令

pwd, ls(-a,-l,-d), cd, mkdir, touch, echo, man, cp, mv, rm(-r,-f)

  • ls: 列出目录中的文件和子目录。可以通过参数（如 -a、-l、-d）来改变其行为。

  • man: 显示命令的手册页。例如，man ls 将显示 ls 命令的手册页。

  • cp: 复制文件或目录。例如，cp source_file destination_file 将复制 source_file 到 destination_file。

  • rm: 删除文件或目录。例如，rm file_name 将删除名为 file_name 的文件。通过参数 -r 或 -f 可以递归删除目录或强制删除文件。

Shell中的特殊字符的作用

单引号，双引号，反引号, $

  • 单引号 (')：关闭变量扩展，不会对变量进行替换。
  • 双引号 (")：允许变量扩展和命令替换。
  • 反引号 (``) ：用于命令替换，Shell 会先执行其中的命令，并将输出作为结果返回。

Bash功能

  • 自动补全功能 <Tab> ：当在命令行中输入部分命令后按 <Tab> 键时，Bash 会尝试自动补全命令或文件名。
  • 查看和执行历史命令：使用上下箭头键可以浏览之前执行过的命令，并使用 !! 可以执行上一个命令
  • 别名：允许用户为常用命令设置别名（alias）

输入输出重定向，管道

  • 重定向输出到文件：使用 > 将输出重定向到文件，如果文件不存在则创建文件；使用 >> 将输出追加到文件末尾。
  • 重定向输入从文件：使用 < 将输入从文件中读取。
  • 管道 (|)：将一个命令的输出作为另一个命令的输入，常用于连接多个命令以实现复杂操作。

通配符的使用

  • *: 匹配任何字符序列（包括没有字符）。
  • ?: 匹配单个字符。这些通配符常用于文件名匹配和过滤操作中。

用户与组管理

root用户和管理员权限（sudo）

  • root用户：root用户是系统中的超级用户，拥有最高权限，可以执行任何操作。
  • sudo权限：允许普通用户通过sudo命令执行一些需要管理员权限的操作，但不需要切换到root用户。

配置文件/etc/passwd

  • /etc/passwd: 存储用户账户信息的文件。
  • /etc/shadow: 存储加密的用户密码信息的文件。
  • /etc/group: 存储组信息的文件。

chpasswd

  • useradd: 用于添加新用户的命令。参数：-g（指定主要组）、-m（创建家目录）、-s（指定登录Shell）。

  • chpasswd: 用于批量为用户设置密码的命令。

  • usermod: 用于修改用户信息的命令，参数：-s（指定登录Shell）。

文件与目录管理

Linux目录结构

  • 根目录 /：Linux文件系统的起点。

  • /etc: 包含系统管理所需要的配置文件和子目录。

  • /var: 用于存储系统运行日志、邮件等动态数据。

  • /home: 用户的主目录，每个用户都有自己的目录。

Linux文件类型分类

  • 普通文件：存储数据的文件。
  • 目录文件：包含其他文件和目录的目录。
  • 链接文件：指向另一个文件或目录的链接。
  • 设备文件：代表系统中的设备，如硬盘、终端等。

stat,ls -l

  • stat: 显示文件的详细信息，包括inode信息。

  • ls -l: 显示文件的详细信息，包括文件类型、权限、所有者等。

cat,more,less,head,tail

  • cat: 显示文件内容。
  • more 和 less: 分页显示文件内容。
  • head 和 tail: 显示文件的前几行或后几行内容。

wc

  • wc: 统计文件的行数、字数和字节数。

find, which

  • find: 在指定目录下查找文件或目录。
  • which: 查找并显示可执行文件的路径。

ln

  • ln: 创建链接文件，包括硬链接和符号链接（软链接）。

tar

  • tar 命令：用于打包和解包文件，可以与 gzip 或 bzip2 结合使用进行压缩或解压缩。选项：-z 使用gzip压缩或解压，-c 创建新的压缩文件，-v 显示详细信息，-f 指定压缩文件名，-x 解压缩。

访问者身份和文件访问权限

  • 访问者身份：用户（u）、组（g）、其他（o）。
  • 访问权限：读（r）、写（w）、执行（x）。
  • 字母表示法：例如，-rwxr-xr-- 表示用户有读、写、执行权限，组和其他用户有读和执行权限。
  • 数字表示法：例如，755 表示用户有读、写、执行权限（7），组和其他用户有读和执行权限（5）。

chown chgrp

  • chown: 改变文件或目录的所有者。例如，chown newuser filename 将 filename 的所有者更改为 newuser。
  • chgrp: 改变文件或目录的组。例如，chgrp newgroup filename 将 filename 的组更改为 newgroup。

chmod

  • chmod: 用于设置文件或目录的访问权限。例如，chmod 755 filename 将 filename 的权限设置为 rwxr-xr-x（即用户有读、写、执行权限，组和其他用户有读和执行权限）。

磁盘存储管理

磁盘分区的命名

  • 概念：将一个物理磁盘划分为多个逻辑部分，每个部分可以独立进行文件系统格式化。
  • 分区的命名：例如 /dev/sda1 表示第一个物理磁盘的第一个分区。
  • Linux分区的类型：Linux分区和Linux Swap分区。

文件系统的概念

  • 概念：文件系统是用于存储和组织数据的机制，它定义了如何存储、检索和管理数据。
  • 分类：ext4、ext3、FAT32等。

fdisk -l

  • fdisk -l: 查看磁盘分区情况。
  • fdisk /dev/sda: 对磁盘 /dev/sda 进行分区操作。

mkfs -t

  • mkfs -t: 根据指定的文件系统类型格式化分区。例如，mkfs -t ext4 /dev/sda1 将在 /dev/sda1 上创建一个ext4文件系统。

mount

  • mount: 将文件系统挂载到目录上，使其可以访问。例如，mount /dev/sda1 /mnt/mydisk 将 /dev/sda1 挂载到 /mnt/mydisk。

umount

  • umount: 卸载已挂载的文件系统。例如，umount /mnt/mydisk 将卸载 /mnt/mydisk 上的文件系统。

sudo file -s /dev/sda1

  • sudo file -s /dev/sda1: 显示 /dev/sda1 上的文件系统类型和块大小。

df -h

  • df -h: 以人类可读的方式显示磁盘空间使用情况。

du -s -h

  • du -s -h: 显示目录的总磁盘使用情况。

  • du 与 ls 查看文件大小的区别：ls 仅显示目录中的文件大小，而 du 显示目录及其子目录的磁盘占用情况。

创建和使用文件系统的步骤

  • 分区：使用 fdisk 或其他工具进行分区。
  • 格式化：使用 mkfs 命令创建文件系统。
  • 挂载：使用 mount 将文件系统挂载到目录上。
  • 使用：通过挂载点访问文件系统中的数据。
  • 卸载：不再需要访问时，使用 umount 卸载文件系统。

软件包管理

APT工具

  • apt: 基于Debian的系统中更现代化的图形界面工具。
  • 更新列表：sudo apt update
  • 升级：sudo apt upgrade
  • 安装：sudo apt install 软件包名
  • 删除：sudo apt remove 软件包名

软件源的概念,配置文件

  • 软件源：定义了可用的软件包的来源，通常配置在 /etc/apt/sources.list 或 /etc/apt/sources.list.d/ 目录下的文件中。

系统高级管理

进程和服务的概念和分类

  • 进程：程序在运行时的一个实例。
  • 服务：后台运行的进程，通常用于启动和管理其他进程。
  • 交互进程：与人机交互的进程，例如文本编辑器或命令行界面。
  • 守护进程：在后台运行的进程，没有用户界面。

前台与后台

  • 前台：用户当前正在与其实时交互的进程。
  • 后台：用户未与其交互的进程，通常在后台运行。

pid

  • PID（进程ID）：每个进程都有一个唯一的ID。

ps,top

  • ps: 显示当前进程的状态。
  • top: 实时显示系统状态和运行中的进程。

进程的启动 挂起 恢复 结束

  • 前台启动：直接运行程序。
  • 后台启动：使用 & 符号在命令末尾，例如 nohup 程序名 &。
  • 挂起：使用 ctrl+z 键挂起当前前台进程。
  • 恢复：使用 fg 命令将挂起的进程移回前台，或使用 bg 命令将进程移到后台。
  • 结束：使用 kill 命令发送信号给进程以结束它。

service, systemctl

  • 服务启动脚本：定义如何启动、停止和重启服务的脚本文件。
  • service: 使用传统的init脚本管理服务。
  • systemctl: 使用systemd工具管理服务和其他系统资源。启动、停止、查看状态、重启服务。

crontab

  • crontab: 用于定期执行任务的工具。
  • at: 在指定的时间执行一次性任务。

日志系统

  • 日志文件的位置：例如 /var/log/ 目录存储了大多数系统日志文件。

  • 查看日志：使用 cat, tail, less 等命令查看日志文件内容。

Shell编程

创建和执行Shell程序

  • 使用文本编辑器创建脚本文件，例如 myscript.sh。
  • 在文件开头添加 #!/bin/bash（或其他shell的路径）作为shebang。
  • 赋予脚本执行权限：chmod +x myscript.sh。
  • 执行脚本：./myscript.sh。

设置和查看环境变量

  • export 命令用于设置环境变量，例如 export MY_VAR="Hello"。
  • 临时环境变量在当前shell会话中有效，关闭会话后失效。
  • 永久环境变量可以添加到 /etc/profile 或 ~/.bashrc 等配置文件中。

PATH环境变量

  • PATH 环境变量指定了系统搜索可执行文件的目录列表。
  • 可以使用 echo $PATH 查看当前的 PATH 设置。

Shell变量

  • 变量名通常以字母或下划线开头，后跟字母、数字或下划线。
  • 变量赋值使用等号（无空格），例如 VAR_NAME="value"。
  • 使用 $VAR_NAME 或 ${VAR_NAME} 进行变量替换。

Shell内部变量（$0、$1、…、$n，以及$#、$*、$@、$?）

  • $0：当前脚本的文件名。
  • $1, $2, …, $n：脚本的第1到n个参数。
  • $#：传递给脚本的参数个数。
  • $*：以单个字符串显示所有传递给脚本的参数。
  • $@：与 $* 类似，但每个参数都是独立的字符串。第九章：网络管理
  • $?：上一个命令的退出状态（0表示成功，非0表示失败）。

Shell数组

  • 定义数组：ARRAY_NAME=("value1" "value2" "value3")。
  • 访问数组元素：${ARRAY_NAME[index]}。

Shell表达式和运算符

  • 算术表达式使用 expr 命令或 $(()) 语法进行计算。
  • 比较运算符用于比较两个值，例如 -eq（等于）、-ne（不等于）、-gt（大于）等。

Shell 逻辑表达式

  • 使用逻辑运算符 &&（与）、||（或）和 !（非）组合条件表达式。

流程控制语句（if, case，for, while, until）

  • if [ condition ]; then ... fi：如果条件为真，执行相应代码块。
  • case "$variable" in ... esac：根据变量的值执行不同的代码块。
  • for variable in list; do ... done：遍历列表中的每个元素并执行代码块。
  • while [ condition ]; do ... done：当条件为真时，重复执行代码块。
  • until [ condition ]; do ... done：当条件为假时，重复执行代码块。

Shell函数

定义函数使用 function_name() { ... } 语法。

调用函数使用函数名后跟参数，例如 function_name arg1 arg2。

网络管理

网络接口概念

  • 网络接口是计算机与网络的连接点，用于数据的发送和接收。
  • 常见的网络接口有以太网接口、Wi-Fi接口等。

查看和设置网络接口（ifconfig）

  • 可以使用ifconfig命令显示当前网络接口的状态、IP地址等信息，并通过参数设置接口的配置。

网络相关的配置文件

  • /etc/hostname：存储主机名。
  • /etc/hosts：存储静态主机名到IP地址的映射。
  • /etc/netplan/01-network-manager-all.yaml：存储网络接口的配置信息。

ping

  • ping命令用于测试与远程主机的网络连通性。
  • 通过发送ICMP回显请求并等待响应，可以确定主机是否可达以及网络延迟情况。

netstat

  • netstat命令用于显示网络连接、路由表、接口统计等网络状态信息。可以通过参数来查看特定的连接状态、监听端口等。
