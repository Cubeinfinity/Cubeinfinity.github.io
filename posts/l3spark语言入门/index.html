1 绪论

RDD

  • 弹性分布式数据集，是spark的基本数据集。

  • RDD的计算主要分为两种：转换计算transformation、执行计算action。

  • 转换计算的计算的结果还是RDD.

    计算流程：输入->转换->转换->…->执行

  • 执行操作结果一般不是RDD，所以执行操作一般是spark的最后一步

    • spark的转换操作是惰性的，只有遇到执行操作，才按顺序执行转换操作。这样可以提高资源利用率。

DAG

  • 有向无环图，是spark计算过程的描述。

Excutor

  • 运行在worker节点上的进程。负责task的运行。

Task

  • 具体的任务计算

Job

stage

一组相互间没有shuffle依赖关系的task组成的集合，也称为TaskSet

  • 窄依赖

数据只在本数据分区内流动，不流出本分区

  • 宽依赖

依赖于多个分区的数据（因此需要等待依赖的分区都计算完成后才能完成计算）

2.4 转换操作

filter

用法：rdd.filter(func)，func要求返回值是布尔值，如果是真，数据传递到新创建的数据集。

要用的数据应该产生于程序之外，而非个人的设计。

wordCount 统计英文单词。

长度过滤

val afterFilter3=rdd1.filter(_.length>4)
afterFilter3.collect

*链式写法

sc.textFile("/home/bigdata3/word.txt").filter(_.length>4).collect

map

用法：rdd.map(func)，将rdd中的每一个元素经过func处理，生成一个新的rdd。例如：每个元素+1

val afterMap=rdd1.map(x=>x+1)
afterMap.collect

sc.textFile("/home/bigdata3/word.txt").map(_+1).collect

flatMap

sc.textFile("/home/bigdata3/word.txt").flatMap(_+1).collect

用法：可以把数据集中各元素之间的分隔去掉，之后的数据按func组合。

e.g. 按照空格分词

sc.textFile("/home/bigdata3/word.txt").flatMap(x=>x.split(" ")).collect

sc.textFile("/home/bigdata3/word.txt").flatMap(x=>x.split(" ")).map(_+"END").collect

reduceByKey

rdd.reduceByKey(func) 应用于(K,V)键值对的数据集时，返回一个新的(K,V)形式的数据集，其中每个值是将每个key传递到函数func中进行聚合后的结果。

处理前期数据：

e.g. 将每个词写成 (词,1) 的形式

val rdd=sc.textFile("/home/bigdata3/word.txt")
rdd.flatMap(x=>x.split(" ")).map((_,1)).collect

rdd.filter(_.length>4).flatMap(x=>x.split("  ")).map((_,1)).collect

reduceByKey统计单词

rdd.filter(_.length>4).flatMap(x=>x.split(" ")).map((_,1)).reduceByKey(_+_).collect

import org.apache.spark.{SparkConf, SparkContext}

object Main {
  def main(args: Array[String]): Unit = {
      val conf=new SparkConf().setAppName("firstSpark").setMaster("local[3]")
      val sc=new SparkContext(conf)
      sc.textFile("/home/bigdata3/word.txt")
        .flatMap(x=>x.split(" "))
        .map((_,1))
        .reduceByKey(_+_)
        .foreach(println)
      Thread.sleep(1000*60*5)
      sc.stop()
  }
}
  • 窄依赖：数据仅在本分区，本程序中的flatMap,map
  • 宽依赖：数据在不同分区有交换（shuffle）。本程序中的reduceByKey

针对DAG：

  • reduceByKey在执行过程中有shuffle操作。
  • stage按shuffle进行分隔。
  • 分区变化：

stage0有两个分区，stage1有两个分区，分区无变化。

*如何让本程序变成两个job？

  • 添加另一个action操作 （saveAsTextFile）

  • import org.apache.spark.{SparkConf, SparkContext}

object Main {
  def main(args: Array[String]): Unit = {
      val conf=new SparkConf().setAppName("firstSpark").setMaster("local[3]")
      val sc=new SparkContext(conf)
      val rdd1=sc.textFile("/home/bigdata3/word.txt")
        .flatMap(x=>x.split(" "))
        .map((_,1))
        .reduceByKey(_+_,3)//此处3！
        rdd1.foreach(println)
        rdd1.saveAsTextFile("/home/bigdata3/sparkOut/4")
      Thread.sleep(1000*60*5)
      sc.stop()
  }
}

创建环境时使用.setMaster(“local[3]")指定了计算使用3个线程，后续如果不指定分区，分区数系统自动优化，数量<=3。

线程一般是按照cpu核心数确定的。

一般有shuffle的转换操作都有一个分区参数。创建RDD的方法也有分区参数。
